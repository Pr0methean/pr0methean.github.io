Class LooperThread

  • java.lang.Object
    • java.lang.Thread
      • io.github.pr0methean.betterrandom.util.LooperThread
  • All Implemented Interfaces:
    Serializable, Cloneable, Runnable
    Direct Known Subclasses:
    RandomSeederThread

    public class LooperThread
extends Thread
implements Serializable, Cloneable

    EXPERIMENTAL. Thread that loops a given task until interrupted (or until JVM shutdown, if it Thread.isDaemon()), with the iterations being transactional. Because of these constraints, it can be serialized and cloned. Subclasses must override iterate() if instantiated without a target Runnable; the only reason this class is concrete is that temporary instances are needed during deserialization.

    Subclasses should override the readResolveConstructorWrapper() method to ensure they are deserialized as a subclass instance.

    iterate()'s body should be reasonably short, since it will block serialization and cloning that would otherwise catch it in mid-iteration.

    Thread state that WILL be restored includes:

    • Thread.getName()
    • Thread.getPriority()
    • getState() == State#NEW
    • getState() == State#TERMINATED
    • Thread.isInterrupted()
    • Thread.isDaemon()

    Thread state that will be restored ONLY if its values are Serializable includes:

    • Thread.getThreadGroup()
    • getUncaughtExceptionHandler()
    • Thread.getContextClassLoader()

    Thread state that will NEVER be restored includes:

    • Program counter, call stack, and local variables. The seederThread will restart.
    • Suspended status (see Thread.suspend()
    • getState() == State#TIMED_WAITING
    • getState() == State#WAITING
    • getState() == State#BLOCKED
    • Thread.getId()
    • Thread.holdsLock(Object)
